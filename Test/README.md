	储存文件的地方共有两个，一个是存在github的服务器上，此外还能存储在本地。

	在本地上修改文件时，还有缓存区的存在。即刚开始做的一切更改都会记录在缓存区。直到用户选择了“更改(commit)”功能，才会将缓存区的更改记录在本地的库中。

	“更改”--用于保存本地的更改

	“同步”--无论是服务器变了还是本地变了，都可以互相传递更改。
